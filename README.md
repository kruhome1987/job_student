# ระบบติดตามชิ้นงานนักเรียน — เชื่อมต่อ Google Sheets ผ่าน Vercel

เอกสารนี้อธิบายวิธีเชื่อม `index.html` (หน้าเว็บที่มีอยู่แล้ว) เข้ากับฐานข้อมูล
Google Sheets ตามโครงสร้างที่ออกแบบไว้ใน `หลังบ้าน.txt` คือ **1 วิชา = 1 ชีต
และบันทึกชิ้นงานเป็นคอลัมน์ต่อไปเรื่อยๆ ทางขวา**

## โครงสร้างที่ได้ (ต่อยอดจากที่ออกแบบไว้)

เอกสารต้นฉบับออกแบบไว้ 2 ส่วน: ชีต `Students` กับชีตรายวิชา แต่หน้าเว็บที่มีอยู่แล้ว
ต้องใช้ข้อมูลเพิ่มเติมต่อชิ้นงาน (ระดับชั้น เป้าหมายห้อง วันกำหนดส่ง คะแนนเต็ม
รายละเอียดงาน) ซึ่งโครงสร้างแบบคอลัมน์-ต่อ-ชิ้นงานเก็บไม่ได้ครบ จึงเพิ่มชีตที่ 3
เข้ามา ผลลัพธ์คือ Spreadsheet 1 ไฟล์จะมี:

1. **`Students`** — รายชื่อนักเรียนทั้งหมด (id, prefix, firstName, lastName, grade, room, no)
2. **`Assignments`** — เมตาดาต้าของชิ้นงานทุกชิ้น (id, subject, title, grade, room, dueDate, maxScore, detail, created) — ชีตนี้เป็นตัวเก็บ "รายละเอียดงาน" ที่ตารางเมทริกซ์อย่างเดียวเก็บไม่ได้
3. **ชีตรายวิชา** (เช่น `คณิตศาสตร์`, `วิทยาศาสตร์`) — ถูกสร้างอัตโนมัติเมื่อมีการสั่งงานวิชานั้นเป็นครั้งแรก โครงสร้างตรงตามที่ออกแบบไว้เป๊ะ: `รหัสนักเรียน | ชื่อ-สกุล | [ชื่องานที่ 1] | [ชื่องานที่ 2] | ...` โดยแต่ละเซลล์เก็บ:
   - ว่าง = ยังไม่ส่ง
   - ตัวเลข = ส่งแล้ว พร้อมคะแนน
   - ข้อความ "ส่งแล้ว" = ส่งแล้ว แต่ไม่เก็บคะแนน

ทุกครั้งที่ครูกด "เพิ่มชิ้นงานใหม่" ระบบจะสร้างชีตวิชานั้น (ถ้ายังไม่มี) และเพิ่ม
คอลัมน์ใหม่ทางขวาสุดให้อัตโนมัติ ตรงตาม Logic ที่ระบุไว้ในเอกสารทุกประการ

## ไฟล์ที่เพิ่มเข้ามา

```
assignment-system/
├── index.html              (ของเดิม แก้เฉพาะส่วน <script> ให้เรียก API แทนข้อมูลจำลอง)
├── package.json
├── .env.example
└── api/
    ├── _lib/sheets.js       (ตัวเชื่อมต่อ Google Sheets กลาง)
    ├── students.js          (GET / POST / DELETE นักเรียน)
    ├── assignments.js       (GET / POST / PUT / DELETE ชิ้นงาน)
    ├── grading.js           (GET รายชื่อสำหรับตรวจงาน + POST บันทึกคะแนน 1 คน)
    ├── grading-bulk.js      (POST ทำเครื่องหมาย "ส่งแล้ว" ทั้งชั้น)
    └── student-view.js      (GET ข้อมูลชิ้นงาน+สถานะของนักเรียน 1 คน สำหรับหน้า Login/ค้นหา)
```

## ขั้นตอนที่ 1: สร้าง Service Account (ทำครั้งเดียว)

1. ไปที่ [Google Cloud Console](https://console.cloud.google.com/) → สร้างโปรเจกต์ใหม่ (หรือใช้ของเดิมที่เคยทำระบบทะเบียนคุมทรัพย์สิน)
2. เปิดใช้งาน **Google Sheets API** ที่เมนู "APIs & Services" → "Library"
3. ไปที่ "APIs & Services" → "Credentials" → "Create Credentials" → **Service Account**
4. ตั้งชื่อ Service Account อะไรก็ได้ → กด "Done"
5. คลิกที่ Service Account ที่สร้าง → แท็บ "Keys" → "Add Key" → "Create new key" → เลือก **JSON** → ดาวน์โหลดไฟล์คีย์เก็บไว้
6. เปิดไฟล์ JSON จะเจอ `client_email` และ `private_key` — 2 ค่านี้ต้องใช้ในขั้นตอนถัดไป

## ขั้นตอนที่ 2: แชร์ Google Sheet ให้ Service Account

1. สร้าง Google Sheet ใหม่ (หรือใช้ไฟล์เดิม) ไว้เป็นฐานข้อมูลของระบบ
2. กด "แชร์" (Share) แล้วเพิ่มอีเมลของ Service Account (`client_email` จากไฟล์ JSON) เป็น **Editor**
3. คัดลอก Sheet ID จาก URL เช่น
   `https://docs.google.com/spreadsheets/d/1AbCdEfGhIjKlMnOpQrStUvWxYz/edit`
   → Sheet ID คือ `1AbCdEfGhIjKlMnOpQrStUvWxYz`

ไม่ต้องสร้างชีตย่อยเอง (`Students`, `Assignments`) ระบบจะสร้างให้อัตโนมัติเมื่อเรียก API ครั้งแรก

## ขั้นตอนที่ 3: ตั้งค่าบน Vercel

1. อัปโหลดโฟลเดอร์นี้ขึ้น GitHub แล้วนำเข้า (Import) เป็น Project ใหม่ใน Vercel
2. ไปที่ Project Settings → Environment Variables ใส่ 3 ค่านี้:

   | ชื่อตัวแปร | ค่า |
   |---|---|
   | `GOOGLE_SERVICE_ACCOUNT_EMAIL` | `client_email` จากไฟล์ JSON |
   | `GOOGLE_PRIVATE_KEY` | `private_key` จากไฟล์ JSON (คัดลอกทั้งก้อน รวม `-----BEGIN/END PRIVATE KEY-----`) |
   | `GOOGLE_SHEET_ID` | Sheet ID จากขั้นตอนที่ 2 |

   ดูตัวอย่างรูปแบบได้ในไฟล์ `.env.example`

3. กด Deploy — Vercel จะรัน `npm install` ให้อัตโนมัติตาม `package.json`
4. เปิดเว็บที่ Vercel ให้มา ระบบจะเรียก `/api/students` และ `/api/assignments` โดยอัตโนมัติ (เพราะ Frontend กับ API อยู่โปรเจกต์เดียวกัน ไม่ต้องตั้งค่า CORS)

## ทดสอบก่อน Deploy จริง (ทางเลือก)

ถ้ามี Vercel CLI ในเครื่อง สามารถรันทดสอบได้ด้วย:
```bash
npm install -g vercel
vercel dev
```
แล้วสร้างไฟล์ `.env.local` ตามตัวอย่างใน `.env.example`

## ข้อควรทราบ / ข้อจำกัดที่ควรปรับต่อ

- **รหัสผ่านครู (PIN)** ยังเป็นค่าตายตัว `1234` ที่ตรวจสอบฝั่ง Frontend เหมือนเดิม (ยังไม่ได้เชื่อมกับ Backend) — ใครก็ตามที่เปิดดูโค้ดหน้าเว็บจะเห็นรหัสนี้ได้ ถ้าต้องการความปลอดภัยจริงจัง ควรทำ endpoint ตรวจสอบ PIN ฝั่งเซิร์ฟเวอร์แทน
- **ถ้าเปลี่ยน "วิชา" ของชิ้นงานที่มีอยู่แล้ว**: ระบบจะลบคอลัมน์เดิมในชีตวิชาเก่า และสร้างคอลัมน์ใหม่ (ว่าง) ในชีตวิชาใหม่ คะแนน/สถานะเดิมจะไม่ถูกย้ายตาม แนะนำให้หลีกเลี่ยงการแก้ "วิชา" ของงานที่ตรวจไปแล้ว
- Google Sheets API มีโควตาการเรียกต่อนาที ถ้าครูหลายคนใช้พร้อมกันจำนวนมากอาจต้องพิจารณาแคชข้อมูลเพิ่มเติมในอนาคต
